# Security analysis

Trace untrusted inputs through parsing, authorization, persistence, command
execution, network access, and output encoding. Check that trust boundaries are
enforced in code rather than implied by callers. Look for credential exposure,
path traversal, injection, confused-deputy behavior, and unsafe defaults.

Apply this as supporting guidance for the primary task. Do not replace the
primary task's output contract or completion tool.
